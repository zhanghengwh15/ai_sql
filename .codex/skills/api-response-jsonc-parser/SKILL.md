# 接口响应 JSONC 解析

## 目标

把接口返回信息整理为可读、可校验的 JSONC 示例。以实际响应为数据依据，以接口文档和上下文为注释依据，不绑定任何业务、接口名称、字段集合或输出目录。

## 工作流

1. 判断输入是原始响应还是接口文档。接口文档可以是纯文本、Markdown、复制的表格、HTML 转文本或混合排版，不依赖固定标题和制表符。
2. 从原始响应、响应示例、返回示例等语义区域定位完整 JSON。去除 HTTP 状态行、请求头和日志前缀；若响应被字符串转义，使用 JSON 解析器还原，禁止用正则改写 JSON 内容。
3. 确定输出范围。保留完整响应时使用 `.`；只输出某个载荷节点时，先检查实际结构，再选择明确的 jq filter，例如 `.data.records`。不要仅凭字段名猜测要丢弃的包裹层。
4. 把有效 JSON 写入临时文件，调用 `scripts/parse_api_response.sh` 生成基础 JSONC。
5. 根据响应参数表、接口说明和上下文，把脚本生成的“含义待确认”改为可靠的业务含义，并补充声明类型、格式、枚举、单位、可空性、加密状态或异常说明。
6. 对照响应示例和响应参数表校验字段值、JSON 类型、数组元素、嵌套层级和记录数量，并报告两者的字段差异。

## 解析接口文档

按以下优先级合并信息：

1. **响应示例**：决定输出根结构、嵌套层级、字段拼写、字段顺序、实际值和实际 JSON 类型。
2. **响应参数表**：提供字段业务含义、声明类型、schema、格式、枚举和约束。
3. **接口描述与上下文**：只用于补充能够可靠确认的整体含义。

严格区分请求与响应：

- 不把请求参数、请求头、路径参数或查询参数写入响应 JSONC，除非响应示例中确实返回同名字段。
- 响应示例中存在但响应参数表未列出的包裹字段仍须保留，并注释“响应参数表未列出”。
- 响应参数表列出但示例未返回的字段默认不补入示例，只在最终答复中列出差异。用户要求完整响应骨架时，才写为 `null` 并注明声明类型和“响应示例未返回”。
- 没有响应示例时，根据响应参数表生成骨架；未知示例值统一使用 `null`，并在注释中写声明类型，禁止伪造业务值。
- 文档类型与示例实际类型不一致时，保留示例值和实际类型，并在注释中同时记录文档声明及差异。
- `string(date)`、`string(date-time)` 等 schema 作为格式写入注释；若示例格式更具体，同时记录示例格式。

## Bash 脚本

```text
parse_api_response.sh INPUT_JSON OUTPUT_JSONC [JQ_FILTER]
```

- `INPUT_JSON`：有效 JSON 文件；传 `-` 时从标准输入读取。
- `OUTPUT_JSONC`：输出文件；传 `-` 时写到标准输出。
- `JQ_FILTER`：可选，默认 `.`；必须恰好产生一个 JSON 值。
- 依赖：Bash、`jq`。

示例：

```bash
<skill-directory>/scripts/parse_api_response.sh \
  /tmp/response.json center/order_response.jsonc '.data.records'
```

脚本负责 JSON 校验、节点提取、格式化、类型注释和原子写入。缺少 `jq` 时明确报告依赖，不要静默安装。

## 数据规则

- 原样保留响应中的键名、值、字段顺序和 JSON 类型。
- 不把数字字符串转换为数字，不把 `0`/`1` 转换为布尔值，不把空字符串转换为 `null`。
- 保留全部数组元素，不只选第一条示例；不要合并结构不同的元素。
- 保留 `null`、空对象和空数组，不根据字段名补默认值。
- JSON 字符串内部即使看起来像 JSON，也默认保持字符串；只有用户明确要求时才展开。
- 不解密、不伪造、不在注释中复述敏感值。敏感信息是否脱敏由用户要求决定，不能静默改变原始值。
- 接口文档与实际响应冲突时，以实际字段名和值类型为准，并在注释中指出差异。

## JSONC 注释

每个字段使用行尾 `//` 注释，优先包含业务含义和实际类型，例如 `String`、`Boolean`、`Number`、`Null`、`Object`、`Array`。能够可靠确认时，再补充：

- 日期时间、编码、精度或单位。
- 枚举值及含义。
- 是否加密、脱敏、可空或可能缺失。
- 文档字段与真实响应的差异。

无法确认的内容写“含义待确认”，不要根据字段名编造业务规则。

## 输出检查

- 输出根类型与所选响应节点一致，不强制为对象或数组。
- 字符串中的引号、反斜杠、换行和 URL 不得被破坏。
- 除行尾注释外，内容符合 JSON 语法：双引号键名和字符串，不使用尾随逗号。
- 最终答复说明输入来源、选择的 jq filter、输出根类型和输出位置，不重复敏感字段值。
